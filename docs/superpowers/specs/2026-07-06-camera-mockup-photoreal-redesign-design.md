# CameraSection 목업 실사 캠 화면 스타일 리디자인

## 배경

`components/CameraSection.tsx`의 우측 미리보기 3종(`DeskCamIllustration`, `CharacterCamIllustration`, room 목업)은 직전 커밋(`ee7b9bd`)에서 flat 벡터 일러스트로 한 차례 개선되었으나, 여전히 "그려진 도형"에 가깝다는 피드백. 새로 추가된 참고 스크린샷(`스크린샷 2026-07-06 오후 2.15.56.png` — 카페에서 아이패드로 화상 스터디를 하는 사진, 아이패드 화면에 참가자 캠 화면 4칸이 그리드로 떠 있음)을 참고해, 실제 화상 캠으로 찍은 듯한 사진적 톤(그라디언트/비네트/베젤)과 멀티 참가자 그리드 구성으로 다시 만든다.

`.refs/` 폴더의 두 스크린샷("차살때" 드라이브 챌린지 랜딩페이지)은 본 작업과 무관하여 참고하지 않는다.

좌측 선택 카드(`cam-card`) 3개, 카드 전환 로직(`CamKey`/`CAM_CARDS`/`CAM_PREVIEW`), 전체 섹션 레이아웃(`cam-layout`)은 이번 범위에서 변경하지 않는다. `components/icons.tsx`의 목업 SVG 컴포넌트 3종과 `app/globals.css`의 관련 스타일만 교체한다.

## 범위

- `components/icons.tsx`: `DeskCamIllustration`, `CharacterCamIllustration` 수정, room 목업용 그리드 마크업(`CameraSection.tsx` 내 인라인 JSX)을 4칸 비디오 그리드로 교체. 필요한 보조 아이콘(작은 실루엣, REC 점 등) 추가.
- `components/CameraSection.tsx`: room 분기(`active === "room"`)의 마크업을 2타일 구조에서 2x2(4타일) 그리드 구조로 교체. desk/char 분기는 컴포넌트 교체만(마크업 구조 변화 없음).
- `app/globals.css`: 공통 "캠 프레임" 톤(비네트, 베젤, REC 점)을 위한 클래스/keyframes 추가, room 그리드 레이아웃(`room-grid`, `room-tile-mini` 등) 신규 스타일 추가, 캐릭터 상태 칩 글래스 스타일 보강.

## 디자인

### 1. 공통 "실사 캠 프레임" 톤

세 목업이 공유하는 시각 언어:

- **베젤**: `.cam-frame` 안쪽에 얇은(1~1.5px) 반투명 테두리 + 살짝의 inner shadow로 "장치 화면 안에 들어있다"는 느낌.
- **비네트**: 각 목업 SVG 최상단에 radial gradient(중앙 투명 → 가장자리 어두운 톤 30~40% 오파시티)를 오버레이 레이어로 추가. 기존 콘텐츠 위에 얹어 사진처럼 가장자리를 눌러준다.
- **렌즈 하이라이트**: 프레임 상단에서 은은하게 밝아지는 linear gradient 얇은 띠 (실제 카메라 렌즈 반사광 느낌), 낮은 오파시티(15% 이하)로 과하지 않게.
- **REC 점**: 프레임 좌상단에 작은 빨간 점 + 얇은 pulse 애니메이션(`@keyframes rec-blink`, opacity 0.4~1 반복). 기존 `.cam-preview-top`의 "LIVE" 텍스트와 중복되지 않도록 위치는 프레임 내부 모서리로 한정.

이 톤은 CSS 클래스 하나(`.cam-frame-realistic` 혹은 기존 `.cam-frame`에 보강)로 구현해 3개 목업에 공통 적용한다.

### 2. 책상/손 캠 (`DeskCamIllustration`)

- 기존 구도(나무 책상 top-view, 스프링노트, 손, 펜 애니메이션) 유지.
- 나무 그라디언트를 2-stop에서 3-stop으로 늘려 중앙에 은은한 하이라이트(빛 반사) 추가.
- 위 "공통 비네트/REC 점"을 이 SVG의 `<defs>` radial gradient로 직접 구현(다른 두 목업과 좌표계가 달라 프레임 크기에 맞춘 별도 정의 필요).
- 손/펜 필기 애니메이션(`pen-write`), 노트 줄 순차 등장은 그대로 유지.

### 3. 캐릭터/상태 표시 (`CharacterCamIllustration`) — 얼굴 클로즈업 합성

"실사 캠에 잡힌 사람의 얼굴만 캐릭터로 가려진" 느낌을 얼굴 클로즈업 구도로 구현:

- 배경: 디테일 있는 장면 대신, 흐린 보케 원 2~3개(반투명, 웜톤 #d9a066/#f2c87b 계열, blur 필터 적용)를 랜덤하게 배치해 "얕은 심도로 찍힌 실내" 느낌만 암시.
- 전경: 기존 곰 캐릭터 얼굴(귀/눈/코/입) 유지. 캐릭터 하단에 반투명 드롭섀도우 타원을 추가해 배경 위에 "얹혀있는" 합성감 부여.
- 기존 `focus-ring` 브리딩 펄스 애니메이션 유지(캐릭터 주변 링).
- 상태 칩("공부 중" + 시계 아이콘)은 `background: rgba` + `backdrop-filter: blur` 글래스 스타일로 업그레이드, 얇은 테두리 추가.

### 4. 캠 필수방 / 캠 선택방 — 2x2 비디오 그리드

기존 2개의 큰 타일을 4개의 미니 비디오 타일 그리드로 교체 (`room-grid` 컨테이너, 2x2):

- **필수방 그룹 (좌측 열, 2칸)**: 캠 On 톤(밝은 배경). 위 칸은 미니 책상캠 썸네일(단순화된 노트+손 실루엣 아이콘), 아래 칸은 흐릿한 인물 실루엣(`PersonDotIcon` 확대 실루엣 + 보케 배경). 위 칸에 브랜드 컬러(`--brand` 등 기존 변수) 선택 테두리 + "캠 필수" 라벨을 그룹 하단에 배치.
- **선택방 그룹 (우측 열, 2칸)**: 캠 Off 톤(어두운 회색조). 두 칸 모두 `CameraOffIcon` + 곰 캐릭터 미니 아바타(`MiniBearIcon`)를 배치해 "카메라 대신 캐릭터로 참여" 표현. 그룹 하단에 "캠 선택" 라벨.
- 4칸 사이 얇은 그리드 라인(구분선, 낮은 오파시티)으로 실제 화상 앱 그리드처럼 분할.
- 그리드 하단(프레임 최하단)에 작은 텍스트로 참가 인원 표기(예: "4명 접속 중")를 추가해 실사 스터디룸 느낌 보강.
- 기존 `room-tile-selected` 애니메이션(select 시 살짝 커지는 효과)은 필수방 그룹의 선택된 칸에 그대로 적용.

## 구현 방식

- `CamKey`/`CAM_CARDS`/`CAM_PREVIEW` 데이터 구조, 카드 클릭 전환 로직은 변경 없음.
- room 목업만 `CameraSection.tsx` 내 인라인 JSX 구조가 2타일 → 4타일 그리드로 바뀌므로 해당 분기 블록을 새 마크업으로 교체.
- desk/char 목업은 `icons.tsx`의 컴포넌트 내부 SVG만 교체, `CameraSection.tsx`에서의 호출부(`<DeskCamIllustration />`, `<CharacterCamIllustration />`)는 그대로 유지.
- 애니메이션은 기존과 동일하게 순수 CSS `@keyframes`로 구현 (추가 라이브러리/이미지 없음, 전부 SVG+CSS).
- 다크 모드: 기존과 동일하게 `.cam-preview` 배경이 고정 다크 톤이므로 별도 라이트/다크 분기 불필요.

## 테스트 계획

- `npm run dev`로 로컬 구동 후 브라우저에서 3개 카드를 각각 클릭해 목업이 올바르게 전환되는지 확인.
- 각 목업의 신규 애니메이션(비네트 정적 오버레이는 애니메이션 없음, REC 점 pulse, 캐릭터 보케+드롭섀도우, room 그리드 select 효과)이 자연스럽게 재생되는지 육안 확인.
- 반응형 브레이크포인트(`.cam-layout` 모바일 스택 지점)에서 특히 4칸 그리드가 찌그러지거나 텍스트가 잘리지 않는지 확인.
